# 개요
- 목표: SSH의 remote tunneling 기능만 분리해 TCP 터널을 구현한다.
- 구성: 공인 IP의 서버가 제어 채널과 외부 공개 포트를 열고, 클라이언트는 서버로 접속해 내부 LAN 대상 호스트로 프록시한다.
- 보안: Ed25519 키로 상호 식별/교환, 공유 비밀에서 HKDF로 세션 키·논스를 파생한다.
- 전송: 모든 프레임을 LZ4로 압축한 뒤 ChaCha20-Poly1305(IETF)로 암호화한다.
- 제약: 서버는 동시에 1개의 클라이언트만 허용한다.
- 서버 실행 환경: Linux CLI 실행을 기준으로 한다.
- 클라이언트 실행 환경: macOS / Linux / Windows 모두에서 동작해야 한다.
- 클라이언트 기능: 여러 개의 터널 구성을 관리하고, 추가/제거를 지원한다.
- 클라이언트 기능: 서버와 일정시간마다 heartbeat 을 주고 받고, 만약 heartbeat 이 끊겼다면 재접속한다.
- 클라이언트 UI: GUI 제공, 각 터널의 접속자 수를 실시간 표시한다.
- 클라이언트 UX: 트레이 아이콘으로 백그라운드 대기(최소화) 가능해야 한다.
- GUI 프레임워크 메모: PySide6 (Qt for Python)는 강력한 크로스플랫폼 지원과 시스템 트레이 연동 등 안정적인 데스크톱 UI 라이브러리다.
- 구현: Python 3.12+ (asyncio) 기반으로 작성하며 Server/Client/Shared 모듈로 분리한다.
- 검증: 가상환경(venv)에서 패키지 설치 후 바로 실행 가능한 샘플 환경과 시나리오를 제공한다.
